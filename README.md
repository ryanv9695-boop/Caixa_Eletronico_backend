O projeto do simulador de caixa eletrônico foi estruturado com a intenção de aprendizado sobre o desenvolvimento backend utilizando comandos simples. A função do sistema é simular as funcionalidades de um caixa eletrônico real, com as principais ações que ele pode ter. Para isso, foram declaradas variáveis do tipo inteiro e double para serem utilizadas dentro do laço de repetição (do-while). É dentro dessa estrutura que tudo acontece: a entrada de dados pelo usuário, as operações matemáticas validadas por estruturas condicionais (if/else e switch/case), e finalmente a saída de dados, imprimindo os resultados na tela.
